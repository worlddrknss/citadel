# Support

## Getting help

Before opening a request, check:

- The project README for setup and usage notes.
- The `docs/` directory for design and implementation details.
- Existing issues for known bugs or feature requests.

## Where to ask

- Bug reports: open a GitHub issue with reproduction steps and expected versus actual behavior.
- Feature requests: open a GitHub issue describing the use case and constraints.
- Security issues: follow the process in `SECURITY.md` and do not file a public issue.

## Response expectations

This is an open-source project with best-effort support. Response times are not guaranteed.
